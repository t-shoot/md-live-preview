# テスト基盤 — 基本設計書

## 概要・システム構成
Markdown Live Preview Editor(VS Code拡張機能)の単体テスト基盤。
DOM非依存の純粋関数を対象に、Vitest(テストランナー)と fast-check(プロパティベーステスト)を使う。

- テストは Node 環境で実行する(`environment: 'node'`)。VS Code本体やWebviewのDOMは使わない。
- Webview の構文木(CodeMirror + `@codemirror/lang-markdown` + GFM拡張)はテスト側でも本番と同じ経路で組み立てて検証する。

## 機能一覧
| 機能 | 説明 |
|---|---|
| `npm test` | 全テストを1回実行して終了する(CI向け) |
| `npm run test:watch` | ファイル変更を監視して再実行する(開発向け) |
| 例示ベーステスト | 具体的な入力→期待する出力を固定するテスト |
| プロパティベーステスト(PBT) | fast-check でランダムな入力を生成し、性質(不変条件・往復変換)を検証するテスト |

## 画面/インターフェース一覧
本機能に画面はない(内部の開発基盤)。CLIコマンド(`npm test` / `npm run test:watch`)のみ。

## 外部連携仕様
- `vitest`(devDependency): テストランナー本体。
- `fast-check`(devDependency): プロパティベーステストのフレームワーク。
- 既存の `tsc --noEmit`(型チェック)・`esbuild`(バンドル)とは独立して動作する。テストファイル(`*.test.ts`)はesbuildのバンドル対象(エントリポイント)に含まれない。
