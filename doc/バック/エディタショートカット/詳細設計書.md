# エディタショートカット — 詳細設計書

## 処理フロー(モジュール/関数単位)

### `computeEmphasisToggle(selected, before, after, from, to, marker)`
純粋関数。`src/webview-editor/emphasisShortcuts.ts` に定義。

判定順序:
1. **選択範囲そのものがマーカーごと含む**(`selected` が `marker` で始まり `marker` で終わり、長さが `marker.length*2` 以上)→ 内側のテキストだけを残す(マーカーを外す)。
2. **選択範囲の直前・直後がちょうど `marker`**(選択範囲外にマーカーがある通常のケース)→ 選択範囲とマーカーをまとめて元のテキストに置き換える(マーカーを外す)。
3. **それ以外**(まだ囲まれていない)→ `marker + selected + marker` に置き換える(マーカーで囲む)。選択なし(空文字列)の場合は空のマーカー対を挿入し、カーソルを内側(2つのマーカーの間)に置く。

戻り値 `EmphasisEdit`:
- `from`, `to`: ドキュメントの置き換え範囲
- `insert`: 置き換え後のテキスト
- `selFrom`, `selTo`: 置き換え後に選択状態にする範囲(カーソル位置を含む)

### `toggleEmphasisCommand(marker: string): Command`
CodeMirrorコマンド。`state.changeByRange` で全てのカーソル/選択範囲それぞれに対して `computeEmphasisToggle` を適用し、まとめて1回のトランザクションとしてdispatchする(複数カーソル編集に対応)。

### キーバインド登録
`src/webview-editor/main.ts` の `keymap.of([...])` 内、`...defaultKeymap` より前に以下を追加:
```ts
{ key: 'Mod-b', run: toggleEmphasisCommand('**') },
{ key: 'Mod-i', run: toggleEmphasisCommand('*') },
```
`keymap.of` は配列の先頭から順にマッチするため、`defaultKeymap` に同名バインドがあっても上書きされない。

### 検索・置換パネル(`@codemirror/search`)— 今回のリリースでは無効化
以下は2026-07-22時点の実装記録。2026-07-23に`src/webview-editor/main.ts`からこの配線(import含む)を削除し、現在は無効。

`src/webview-editor/main.ts` の `createExtensions()` に `search()` 拡張を追加し、`keymap` に `...searchKeymap`(既定バインド一式)を追加していた。

`@codemirror/search` の既定パネルは検索欄・置換欄を最初から両方含む1つのUIであり、別々の「置換パネル」は存在しない。

#### `openReplacePanel(view: EditorView): boolean`(削除済み)
`src/webview-editor/main.ts` に定義していたUI関数(DOM操作のみ、純粋関数ではない)。
1. `openSearchPanel(view)` でパネルを開く。
2. `view.dom.querySelector('.cm-search input[name="replace"]')` で置換欄の`<input>`を取得し、`.focus()`する。

キーバインド登録: `{ key: 'Mod-h', run: openReplacePanel }` を `keymap` 配列に追加していた。

**関連CSS**: `media/webview-editor-theme.css` にVS Code風フローティングデザインへ作り直したスタイル一式(`.cm-panel.cm-search`等)が残っている。「現在未使用」の注記付きで、削除はしていない。`main.ts`に上記配線を戻せばそのまま有効になる。

**関連依存**: `package.json` の `@codemirror/search` も、再開時の手間を減らすため残置している。

## 既知の制約
- 太字(`**`)と斜体(`*`)が入れ子になった状態(例: `***text***`)は、各ショートカットが自分のマーカーのみを見て判定するため、賢い相互変換はしない。
