# 画像の貼付・ドラッグ&ドロップ — 基本設計書

## 概要・システム構成
Markdown Live Previewのエディタ画面で、クリップボードからの画像貼付、またはファイルのドラッグ&ドロップを検知し、画像ファイルを保存して`![](...)`記法を自動挿入する機能。

## 機能一覧
| 機能 | 説明 |
|---|---|
| 画像の貼付 | クリップボードに画像がある状態で貼付すると、画像を保存し記法を挿入 |
| 画像のドロップ | 画像ファイルをエディタ上にドラッグ&ドロップすると、同様に保存・挿入 |
| 保存先 | 開いている`.md`ファイルと同じ場所の`assets/`サブフォルダ(自動作成) |
| ファイル名の自動生成 | `image-<タイムスタンプ>.<拡張子>`。衝突時は`-1`,`-2`...を付与 |

## 画面/インターフェース一覧
Markdown Live Preview のエディタ画面(Webview)内で完結する。専用のダイアログ等はない。

## 外部連携仕様
- Webview→ホスト: `EditorToHostMessage`に`pasteImage`(位置・MIMEタイプ・base64データ)を追加。
- ホスト→Webview: `HostToEditorMessage`に`setCursor`(挿入後のカーソル位置)を追加。
- 実際のファイル保存・ドキュメント編集はホスト側(`vscode.workspace.fs`, `vscode.WorkspaceEdit`)で行う。
