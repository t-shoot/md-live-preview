# アウトライン(見出し一覧) — 詳細設計書

## 処理フロー(モジュール/関数単位)

### `extractHeadings(text: string): HeadingItem[]`(`src/shared/headings.ts`)
純粋関数。行単位でATX見出し(`#`〜`######`、1〜6個)を判定する。
- フェンスコード(```` ``` ```` / `~~~`)の内側にある行は判定対象外にする(開始・終了マーカーの種類と長さを追跡)。
- `HeadingItem = { level, text, line }`。`line`は1始まりの行番号(CodeMirrorの`doc.line(n)`にそのまま渡せる)。
- **既知の制約**: 行頭に半角スペースでインデントされた見出し(CommonMark仕様上は3つまで許容)は非対応。単純化のため。

### アクティブなプレビューの検出(`MarkdownLivePreviewProvider.findActiveSession()`)
1. `vscode.window.tabGroups.activeTabGroup.activeTab?.input` を取得。
2. `vscode.TabInputCustom` かつ `viewType === MarkdownLivePreviewProvider.viewType` であることを確認(`extension.ts`の`getActiveCustomEditorUri()`と同じ判定)。
3. そのタブのURIと一致する`DocumentSyncSession`を`sessions`から探す。

### `MarkdownLivePreviewProvider.getActiveHeadings()` / `jumpToActiveHeading(line)`
- `getActiveHeadings()`: アクティブなセッションのドキュメントテキストを`extractHeadings`に渡して返す。アクティブなセッションが無ければ`undefined`。
- `jumpToActiveHeading(line)`: アクティブなセッションの`jumpToLine(line)`を呼ぶ(何もしない場合はセッションが見つからない=何も起きない)。

### `DocumentSyncSession` への追加
- `getDocument(): vscode.TextDocument` — 既存の`document`を返す。
- `jumpToLine(line: number): void` — `{type:'jumpToLine', line}`をWebviewへpost。

### `OutlineViewProvider`(`src/sidebar/OutlineViewProvider.ts`)
- `vscode.window.tabGroups.onDidChangeTabs` と `vscode.workspace.onDidChangeTextDocument` を購読し、150msデバウンスして`refresh()`を呼ぶ。
- `refresh()`: `editorProvider.getActiveHeadings()`の結果を`update`(見出しあり)または`noDocument`(アクティブなプレビューなし)としてWebviewへpost。
- Webviewからの`jumpToHeading`メッセージは`editorProvider.jumpToActiveHeading(line)`に中継。

### `src/webview-editor/main.ts` の `jumpToLine` ハンドリング
`view.state.doc.line(message.line).from` を選択位置にして`scrollIntoView: true`でdispatchし、`view.focus()`する。行番号が範囲外(`< 1`または`> doc.lines`)の場合は何もしない(見出し抽出後にドキュメントが外部から変更され行数が変わった場合の安全策)。

### Webview本体(`src/webview-outline/main.ts`)
- 見出しをレベルごとにインデントした`<ul>`で描画。
- 各項目はクリックまたはEnter/Spaceキーで`jumpToHeading`をpost(`role="button"`、`tabindex="0"`でキーボード操作にも対応)。
- 見出しが0件、またはアクティブなプレビューが無い場合は案内文を表示。

## データ設計(テーブル・DB定義)
該当なし。

## エラーハンドリング仕様
- `jumpToLine`はメッセージの`line`がドキュメントの行数を超えていれば何もしない(クラッシュしない)。
- `OutlineViewProvider.refresh()`はビューが破棄されている(`this.view`が`undefined`)場合は何もしない。
